# Developer Portfolio

GitHub Pages 배포를 염두에 둔 프로젝트 중심 개발자 포트폴리오입니다.

## 구성

- `index.html`: 페이지 구조
- `styles.css`: 반응형 디자인과 스타일
- `script.js`: 대표 프로젝트 데이터와 카드 렌더링

## 수정할 곳

1. `index.html`의 이름, 소개 문구, 연락처, GitHub/LinkedIn 링크를 본인 정보로 변경합니다.
2. `script.js`의 `projects` 배열에서 프로젝트명, 설명, 기술 스택, 데모 링크, GitHub 링크를 변경합니다.
3. GitHub 저장소의 Pages 설정에서 배포 브랜치를 선택합니다.

## GitHub Pages

정적 사이트이므로 별도 빌드 없이 배포할 수 있습니다.

저장소 Settings > Pages에서 `Deploy from a branch`를 선택하고, `main` 브랜치의 root를 지정하면 됩니다.
